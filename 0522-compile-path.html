TileLang JIT Kernel 编译路径学习笔记

从 Python DSL / PrimFunc 到最终 CUDA 源码与可执行 kernel 的主路径梳理。

一、整体路径

Python DSL / @tilelang.jit
  -> PrimFunc / IRModule
  -> tilelang.compile / JITKernel
  -> tilelang.lower
  -> LowerAndLegalize + OptimizeForTarget
  -> host/device IR split
  -> target.build.tilelang_cuda(_without_compile)
  -> CodeGenTileLangCUDA 生成 CUDA C++
  -> adapter 编译、加载、封装调用

可以把编译链路分成四层：前端生成 TIR、JIT 调度编译、TileLang/TVM pass 降低优化、CUDA codegen 与运行时封装。

二、前端入口：Python 代码到 PrimFunc

@tilelang.jit

装饰器入口在 tilelang/jit/__init__.py。它把 Python 函数包装成 JITImpl，支持 lazy 和 eager 两种模式。

tilelang.compile

直接编译已有 PrimFunc。入口同样在 tilelang/jit/__init__.py，之后进入缓存和 JITKernel。

T.Kernel

建立 kernel launch frame，记录 grid/block/thread 绑定，位置在 tilelang/language/kernel.py。

lazy 模式通常是用户显式返回 @T.prim_func 产生的 PrimFunc；eager 模式则通过 tilelang/language/eager/builder.py 的 builder 在执行 Python 函数时构造 TIR。

三、JITKernel：编译调度与 adapter 选择

tilelang.compile 会经过 tilelang/cache/kernel_cache.py 的缓存逻辑，cache miss 后创建 JITKernel。

核心方法是 JITKernel._compile_and_create_adapter，位置在 tilelang/jit/kernel.py。它负责：

  • 规范化 target、execution backend、pass config 和 compile flags。
  • 创建 tvm.transform.PassContext。
  • 调用 tilelang.lower(...) 得到 CompiledArtifact。
  • 根据 backend 创建 TVMFFIKernelAdapter、NVRTCKernelAdapter、CythonKernelAdapter 等。
注意：tvm_ffi backend 会在 tilelang.lower 阶段启用 host codegen 和 device compile；nvrtc、cython 等路径通常先拿 CUDA source，再由 adapter 负责编译/封装。

四、lower：PrimFunc 到 host/device IR

tilelang.lower 位于 tilelang/engine/lower.py。主函数会先调用 lower_to_host_device_ir，完成 IR 降低和 host/device 拆分。

  1. 如果输入是 PrimFunc，先包装成 IRModule。
  2. 解析 target 与 target_host。
  3. 执行 PreLowerSemanticCheck。
  4. 执行 LowerAndLegalize。
  5. 执行 OptimizeForTarget。
  6. 根据 CallingConv.DEVICE_KERNEL_LAUNCH 过滤出 host_mod 和 device_mod。

五、主要 pass pipeline

LowerAndLegalize

位置：tilelang/engine/phase.py。这一阶段更偏前端语义合法化和高层 Tile IR 降低。

  • BindTarget：把 target 绑定到模块。
  • LegalizeNegativeIndex、VerifyParallelLoop、InjectAssumes、Simplify。
  • LayoutReducer、LayoutInference：处理 fragment/shared memory 布局。
  • ProducerConsumerWarpSpecialized、PipelinePlanning、InjectSoftwarePipeline。
  • LowerTileOp：把高层 tile op 降到更低层操作。
  • LowerL2Persistent、LegalizeVectorizedLoop、LegalizeSafeMemoryAccess、LowerAccessPtr。

OptimizeForTarget

位置同样在 tilelang/engine/phase.py。这一阶段更偏 CUDA/TIR 后端优化和设备函数准备。

  • LowerSharedTmem、LowerSharedBarrier、PlanAndUpdateBufferAllocationLocation。
  • FlattenBuffer、VectorizeLoop、StorageRewrite、UnrollLoop。
  • LowerThreadAllreduce、LowerLDGSTG、LowerHopperIntrin。
  • AnnotateDeviceRegions、SplitHostDevice：为 host/device 拆分做准备并完成拆分。
  • MergeSharedMemoryAllocations、ThreadSync、MakePackedAPI、LowerDeviceKernelLaunch。
  • PersistThreadblock：CUDA threadblock 持久化相关变换。

六、device IR 到 CUDA source

tilelang.lower 对 device_mod 调用 codegen：

  • device_codegen：生成 CUDA source，并通过 nvcc callback 编译成 cubin，主要用于 tvm_ffi。
  • device_codegen_without_compile：只生成 CUDA source，主要用于 nvrtc、cython 等 adapter 后续处理。

CUDA target 最终调用 C++ FFI 注册函数：

target.build.tilelang_cuda
target.build.tilelang_cuda_without_compile

注册和入口在 src/backend/cuda/codegen/rt_mod_cuda.cc。它会创建 CodeGenTileLangCUDA，遍历 device PrimFunc，调用 cg.AddFunction(...)，最后通过 cg.Finish() 得到完整 CUDA C++ 源码。

七、CUDA codegen 的关键点

主要实现文件：src/backend/cuda/codegen/codegen_cuda.cc。

  • PrintFuncPrefix：输出 extern "C" __global__。
  • PrintExtraAttrs：从 thread_extent 提取线程数，生成 __launch_bounds__。
  • AddFunction：把 device PrimFunc 打印成 CUDA kernel 函数。
  • Finish：拼接 CUDA 头文件、TileLang template 头文件和最终源码。

CUDA template 和 intrinsic 相关文件集中在：

  • src/tl_templates/cuda/：copy、gemm、reduce、debug、MMA/WGMMA/TCGEN05 等模板。
  • src/backend/cuda/op/：CUDA op lowering 和指令选择相关实现。

八、最终编译、加载和调用

生成的 CUDA source 会保存在 CompiledArtifact.kernel_source，adapter 再把它封装为 Python 可调用对象。

tvm_ffi 路径

adapter 在 tilelang/jit/adapter/tvm_ffi.py。这一条路径使用 TVM runtime module，host module 会 import device module，最终通过 TVM executable 调用。

nvrtc 路径

adapter 在 tilelang/jit/adapter/nvrtc/adapter.py，编译器在 tilelang/jit/adapter/nvrtc/libgen.py。

  • 使用生成的 CUDA source 写出临时 .cu。
  • 通过 NVRTC 编译出 .cubin。
  • 生成 Python launcher .py。
  • 通过 CUDA Driver API 加载 cubin 并取 kernel handle。

launcher 生成逻辑主要在 tilelang/jit/adapter/wrapper.py，负责生成 kernel launch、dynamic shared memory、cluster launch、TMA descriptor 等 host 侧调用代码。

九、关键文件索引

  • tilelang/jit/__init__.py
    JIT decorator、compile、lazy/eager 调度。
  • tilelang/jit/kernel.py
    JITKernel，调用 tilelang.lower 并选择 adapter。
  • tilelang/cache/kernel_cache.py
    kernel 编译缓存，cache miss 后构造 JITKernel。
  • tilelang/language/kernel.py
    T.Kernel 和 T.CUDASourceCodeKernel 前端 launch frame。
  • tilelang/language/eager/builder.py
    eager JIT builder，把 Python DSL 执行过程转成 TIR。
  • tilelang/engine/lower.py
    lower 总入口、host/device IR split、device/host codegen 调度。
  • tilelang/engine/phase.py
    主要 pass pipeline：LowerAndLegalize 与 OptimizeForTarget。
  • src/transform/split_host_device.cc
    host/device 拆分相关 C++ pass。
  • src/backend/cuda/codegen/rt_mod_cuda.cc
    CUDA build FFI 注册点：target.build.tilelang_cuda*。
  • src/backend/cuda/codegen/codegen_cuda.cc
    CUDA C++ 代码生成器 CodeGenTileLangCUDA。
  • src/tl_templates/cuda/
    TileLang CUDA template 头文件。
  • tilelang/jit/adapter/nvrtc/
    NVRTC 编译、加载 cubin、Python launcher。
  • tilelang/jit/adapter/tvm_ffi.py
    TVM runtime 路径的执行适配。

十、建议阅读顺序

  1. 先读 tilelang/jit/__init__.py 的 jit 和 compile。
  2. 再读 tilelang/jit/kernel.py 的 _compile_and_create_adapter。
  3. 接着读 tilelang/engine/lower.py 的 lower_to_host_device_ir 和 lower。
  4. 重点看 tilelang/engine/phase.py 两个 pipeline。
  5. 最后看 rt_mod_cuda.cc 与 codegen_cuda.cc，确认 CUDA source 是怎么打印出来的。
